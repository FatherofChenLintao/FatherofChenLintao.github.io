目前所学所有攻击手法均需要有输入漏洞
本题有两个入口
第二个入口可产生数据溢出
char v2[32]; // [esp+10h] [ebp-74h]
__isoc99_scanf("%s", v2);
v2真正大小只有：[esp+10h] [ebp-22h]，即32个字节

在从后往前分析一下：

  1. 找到一个system函数可用，但没有关键字符串可用，或许可以自己构造

  2. 又找到一个函数，几乎可以确定为目标函数
    `int sub_80486CC()
    {
    char s; // [esp+1Eh] [ebp-3Ah]

    snprintf(&s, 0x32u, "cat %s", "./flag");
    return system(&s);
    `}``
    目标函数首地址为：0x080486CC

  3. 找到一个反编译失败函数，地址：0x8048A61

    call eax是突破口
